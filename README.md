# japanese-tech-writing-ex

生成 AI に日本語の技術文書（書籍の章、技術記事、解説文）を**書かせる／推敲させる／公開前に採点する**ための Claude スキルです。名前の `-ex`（extended）は、源流の規範を全文収録したうえで派生元の運用を継承した拡張版であることと、上流・同名の派生スキルと**共存できる別の識別子**であることを示します。`k16shikano` 氏の「日本語技術文書の文章規範」を基盤に、論理的で誠実な日本語テクニカルライティングを生成し、LLM が陥りがちな「ちゃんと書いている感」だけの文章（AI 臭）を抑制します。

## 成り立ち

このスキルは既存の二つの著作物を組み合わせ、欠落を補って拡張したものです。詳しくは [NOTICE](NOTICE) と [LICENSE](LICENSE) を参照してください。

- **規範本文** ([reference/rules.md](reference/rules.md)) — 源流である k16shikano 氏の gist（Unlicense）の全文を、**要約せず**収録しています。源流が単一ファイルに inline していた「見出しの付け方」「読者への誠実さ」を含む全観点と、すべての悪い例／良い例を欠落なく写しました。
- **運用・補助資料** — モード運用・チェックリスト・採点ルーブリック・AI 臭語彙リストの構成は、[hikimay/japanese-tech-writing](https://github.com/hikimay/japanese-tech-writing)（MIT）を継承しています。語彙リストはさらに [iKora128/stop-ai-slop-jp](https://github.com/iKora128/stop-ai-slop-jp)（MIT）に由来します。

この版の主眼は、**源流の規範を一字も落とさないこと**と、**運用の足回り（書く／直す／採点する手順）を備えること**の両立です。


## 使い方

Claude Code 等のスキルディレクトリ（例: `~/.claude/skills/`）にこのリポジトリを配置すると、次回セッションから利用できます。

- 「技術記事を書いて」「この文章を推敲して」「AI 臭を消して」「公開前チェックして」などのトリガーで起動します。
- 推敲は「主張・厳密さ → 主体性 → 構造 → 語彙 → 整形」の順で直します。
- 公開前チェックは 6 軸（主張・厳密さ・主体性・具体性・簡潔さ・語り）を各 10 点で採点し、合格ラインは 42/60 です。
